# v0-05d10 Goal: Abstract Policy Key

## Strategic Pre-Loop Audit Finding
v0-05d9 smoke test revealed: live hit_rate = 1.6% vs holdout hit_rate = 62.7%.
The policy table fires on only 7/439 eligible UNKNOWN_0 decisions in live play.
Root cause: keys encode concrete card IDs (e.g. `12|13|14`) that are dataset-specific.
In live games vs other agents, different card ID combinations appear → near-total miss.

## Hypothesis
Replacing the concrete `option_signature` (`12|13|14|END|NO`) with an abstract form
(`N3|END|NO`) that encodes `n_numeric_options + keyword flags` will:
1. Increase live hit_rate from 1.6% to 20%+ (primary target)
2. Maintain or improve holdout winner_margin (>= 0.084, baseline v05d9)

## What Is Tested
- Abstract key: `_to_abstract_option_sig()` in notebook, `_unknown0_policy_abstract_sig()` in main.py
- Key format: `{N0|N1|N2|N3p}|{END?}|{YES?}|{NO?}|{NUMBER?}||{deck}||{ko}||{turn}`
- USE_ABSTRACT_OPTION_SIGNATURE=True in both notebook and main.py

## What Is NOT Changed
- MC return weighting (gamma=0.98, kept from v05d9)
- MLP submission: OFF
- Data: 2026-06-24 only (latest)

## Success Criteria
- Primary: holdout winner_margin >= 0.081 (v05d7 baseline) AND live hit_rate > 10%
- Strong success: winner_margin >= 0.084 (v05d9) AND live hit_rate > 20%

## Causal Assumptions
- Abstract sig covers same game situations as concrete sig (it's a superset)
- The table can still learn meaningful actions with fewer unique keys
- Live game UNKNOWN_0 situations have predictable n_numeric/keyword patterns

## Unexpected Finding Protocol
- If hit_rate improves but winner_margin drops: abstraction loses precision → next: keep abstract but increase min_table_acc
- If hit_rate stays low: abstract sig still doesn't match runtime → investigate _unknown0_policy_option_type_identifiers output
